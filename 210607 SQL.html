B조 인원들은 최종 프로젝트를 위해 아이디어 회의를 하던 도중 각자의 개인 인적 사항, 취미, 아이디어들 일목 요연하게 정리할 필요성을 느껴
테이블을 이용한 정리의 필요성을 느꼈다. sql문장을 이용하여 다음을 구현하여라.

1. PERSON_INFO / HOBBY / PROJECT 테이블을 생성하여라. PERSON_INFO 테이블에는 NO, NAME, AGE, HOBBY, INTEREST, PROJECT_IDEA 여섯개의 컬럼이 존재하며 HOBBY 테이블에는 H_NAME(취미 이름), H_CATA(활동성 여부(I / O) 둘 중 하나만 받는다.) 두 개의 컬럼이 존재하며 PROJECT_IDEA 테이블에는 IDEA(아이디어), REQUIRMENT_LV(구현 난이도), ESSENTIAL_SKILLS(필요 기술), ESTIMATED_TIME(개발 시간) 세개의 컬럼이 존재한다.

모범 답안 DROP TABLE PERSON_INFO; CREATE TABLE PERSON_INFO( NO NUMBER(3), NAME VARCHAR2(20), AGE NUMBER(3), HOBBY VARCAHR2(20), INTEREST VARCHAR2(20), IDEA VARCHAR2(20) );

2. job_history 테이블에서 업무 종료일자(end_date)가 06년 이전인 데이터 중 부서코드(department_ID), 시작일(start_date), 종료일(end_date)를 출력하시오.

모범 답안 select department_id, start_date, end_date from job_history where end_date < '06/01/01'; DEPARTMENT_ID START_DATE END_DATE ----------------------- ----------------- -------------- 110 97/09/21 01/10/27 110 01/10/28 05/03/15 90 95/09/17 01/06/17

3. locations 테이블에서 알파벳 s(대소문자 구별 없이)가 포함된 거리 주소(street_address), 우편번호(postal_code), 도시(city)를 출력하시오.
